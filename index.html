欢迎来到MengQi的世界

大家好，这是我的第一篇博客。终于，我也是有博客的人了

  1. 为什么想搭博客

    作为一名优（拖）秀（延）的程序猿，我一直想有一个博客。总觉得搭个人博客是一个很牛掰的事情，这么牛掰的事情做起来一定很难。但是今天真正做起来感觉还可以。果然是万事开头难吧。从今天开始，我也有了自己的博客。开心ing。

  2. 搭博客遇到的问题记录

    首先要感谢记录这几篇文章博主，跟着做就好了

    • 韦阳的博客
    • https://juejin.cn/post/6844903937791770637
    • https://segmentfault.com/a/1190000017986794

    （1）仓库名字

    因为github是我之前就创建的账号，搭博客创建仓库的时候，我很蒙，仓库名字一定要用自己的名字是什么，然后我先创建了一个以我姓名拼音的仓库，一直感觉怪怪的。后面看了别文章才知道仓库名字是用户名，就是创建github账号的用户名。避坑~

    （2）hexo 部署提示deploy done:git成功，但是服务端没有产生文件

    这个问题真的绝绝子，困扰了我好久，俩小时吧。我觉得一切都很正常，跟着大神的步骤做的。结果一直推不上去。然后我又看了好几眼github，终于发现我有俩分支，我一直推在master分支，但是界面展示一直是main分支。

    纳了闷了，我什么时候做了这个main分支（我不是很会操作git，一知半解的）。我就想到了操作步骤有一个我当时没做，当时为啥没做，忘了。我回去在【Settings】【Pages】【Source】将分支改正master，输博客网址页面也显示了。搜索后知道page仓库的master分支用来存放网站文件的，这是GitHub Page的要求。

    （3）图片加载不出来

    我在本地用Typora软件写的md文件，他会自动新建一个和md文件同名的.assets目录。但是hexo生成的是一个同名的目录

    • hexo生成的目录【这是正确的，路径千万不要错，不然图片打死也出不来】

    • md文件图片保存的目录

    上面只记录了报错的，加载图片的过程如下。

    • 下载插件：npm install hexo-asset-image --save
    • 修改全局文件：在根目录文件【_config.yaml】找到【post_asset_folder】，将值改为【true】
    • 之前新建的文章自己建一个同名的目录，随后新建的文章自己会自动新建一个同名的目录

  3. 博客想记录什么

    我的废话发在别的地方了。博客我想记录知识点类的，有助于工作技能提升，兴趣爱好之类的。也是学习的一个过程吧。
